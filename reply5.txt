The draft seed now maps `editing.unit` through `normalizeTimeUnit` (new `canonicalTimeUnit` helper), falling back to "min", so the time-unit `<select>` always receives a valid hr/min/sec value. This covers both a non-canonical stored unit ("hr/night", a corrupted doc) and the case of editing a plain numeric metric (e.g. unit "kg") and toggling Format to Time. Added tests for both scenarios.
